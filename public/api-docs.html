WhatsApp Unified API Layer (RPC)

Highly concurrent, one-endpoint payload-based architecture built to scale to thousands of users instantly.

Target URL: POST /api/whatsapp
⚠️ Architecture Note:

1 Unified URL: You only ever make POST requests to /api/whatsapp! Tell the server what to do by providing an "action" property in your JSON body.

Single Tenant: You do NOT need a session ID! The system automatically hooks you to a singleton "default" bot layer.

Multi Tenant Isolation: If 4 employees are using this simultaneously, attach a "sessionId": "personA" to your JSON body payload to ensure environments never overlap!
"action": "start"
Spins up a headless Chromium container strictly isolated for you. You must run this action ONCE before utilizing other actions.
Request Body Example
{
    "action": "start",
    "sessionId": "personA" // Optional for multi-tenant
}
"action": "status"
Returns the connection state for your active session background worker.
Request Body Example
{
    "action": "status",
    "sessionId": "personA"
}
"action": "qr"
Retrieves the QR code base64 image (if the given session is awaiting to be scanned). Note that puppeteer boot takes up to 8 seconds so the QR may jump asynchronously.
Request Body Example
{
    "action": "qr",
    "sessionId": "personA"
}
"action": "pair"
Alternative 8-digit OTP string pairing login using a phone number.
Request Body Example
{
    "action": "pair",
    "sessionId": "personA",
    "phone": "919876543210"
}
"action": "send"
Triggers a message dispatch over your specific linked socket infrastructure.
Request Body Example
{
    "action": "send",
    "sessionId": "personA",
    "phone": "919876543210",
    "message": "Hello from the new RPC architecture!"
}
"action": "send-receipt"
Transmits a Base64-encoded file (like PDF, Image, etc) directly via node protocol buffers.
Request Body Example
{
    "action": "send-receipt",
    "sessionId": "personA",
    "phone": "919876543210",
    "base64Data": "JVBERi0xLjQK...",
    "mimetype": "application/pdf",
    "filename": "invoice_123.pdf",
    "caption": "Here is your invoice!"
}
"action": "send-bulk"
Background throttled mass-messaging array system that dodges WhatsApp rate limiting algorithms.
Request Body Example
{
    "action": "send-bulk",
    "sessionId": "personA",
    "messages": [
        { "phone": "1234567890", "message": "Offer A" },
        { "phone": "0987654321", "message": "Offer B" }
    ]
}
"action": "messages"
Extracts recent chat history from a thread.
Request Body Example
{
    "action": "messages",
    "sessionId": "personA",
    "phone": "919876543210",
    "limit": 50
}
"action": "logout"
Gracefully unbinds the local socket instance, dumps ram pointers, and kills matching background node.js threads for the specific session ID specified in the payload.
Request Body Example
{
    "action": "logout",
    "sessionId": "personA"
}
"action": "clear-all"
GLOBAL EMERGENCY NODE RESET. Sends a master kill signal traversing all session maps locally, terminates any dangling chromium memory hooks, and dumps physical disk auth cache folders out of root.
Request Body Example
{
    "action": "clear-all"
}
